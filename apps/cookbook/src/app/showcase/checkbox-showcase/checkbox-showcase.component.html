A Checkbox is a control that allows the user to select options. Checkboxes are used for multiple choices, not for mutually exclusive choices. Each checkbox works independently from other checkboxes in a list, therefore checking an additional box does not affect any other selections.

Types

There are two types of Checkboxes:

  1. Attention Level 1: Use as a binary choice for a standalone check box
  2. Attention Level 2: Use in a list of check boxes, where multiple choices can be selected

Sizes

The small and medium sizes apply to the clickable/tapable area (highlighted in the examples below) while the checkbox itself remains the same size.

The extra small size applies to both click-area and checkbox, but should be used with caution.

The extra small size is mainly intended for use in expert systems or information dense interfaces where the reduced size is a necessity but note that it comes at the cost of degraded accessibility.

Indeterminate state

The most common use-case for an indeterminate checkbox is as a parent to a group of nested checkboxes. If all of the children are checked, the parent checkbox is checked, and if all are unchecked, the parent checkbox is unchecked. If any one or more of the children have a different state than the others, the parent checkbox is indeterminate.

In this pattern, the parent checkbox should also provide functionality to select/deselect all children. See below for a reference implementation of this pattern.

Angular Forms

The checkbox can be used directly in template driven and reactive forms . To use checkbox in a reactive form, bind your form control to <kirby-checkbox> with simple data binding:

<kirby-checkbox [formControl]="yourFormControl"> // or simply: <kirby-checkbox formControlName="yourFormControlName">

Accessibility

Always provide a label for the checkbox to inform screen reader users what the checkbox is controlling.

When using the checkbox without a visible label, set the aria-label property instead.
