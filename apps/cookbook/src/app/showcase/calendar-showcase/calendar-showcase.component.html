The calendar is a picker interface that displays a date grid with month navigation, for users to select a single date.

Calendar in card

The calendar should in most use-cases be placed inside a card to stand out from the canvas.

By default any date is selectable for the user but various configurations can be applied to restrict what dates users can select. For instance, the calendar can be configured with min and max dates, disable weekends, disable specific dates and more.

This example showcases the many different options, and will also allow you to play around with setting the selectedDate programmatically.

Calendar with year navigator

A year-navigator can be shown in the calendars header to provide quick navigation between years.

Calendar without background

The calendar comes with minimal styling out of the box and we recommend to place it in a kirby-card .

In some rare use-cases it can be necessary to place the calendar on a different background or directly on canvas, in which case the calendar can be used without any surrounding card.

Accessibility

The calendar uses a table to provide an easily navigable grid of dates. Screen-reader users can navigate using standard table navigation commands.

When using the calendar on another background than a white kirby-card , additional care should be taken by implementers to ensure proper contrast and readability of the date grid and controls.

Keyboard support

The calendar has full keyboard support to make it easier to navigate and select dates.

The following keys can be used to move the currently focussed date:

  • Arrow keys
    • ← and → moves focus to previous and next day
    • ↑ and ↓ moves focus to same weekday of previous and next week
  • Home and End moves focus to first day (Monday) and last day (Sunday) of the current week
  • PageUp and PageDown moves focus to same day of previous and next month
  • Shift + PageUp and Shift + PageDown moves focus to same day and month of previous and next year

When focussed, a date can be selected with Enter ↵ and Space.

Specifying locale

The locale of the calendar (i.e. day and month names) can be set by providing a LOCALE_ID . The calendar currently supports LOCALE_IDs da , de , en-GB and en-US and it defaults to en-GB if none is provided.
