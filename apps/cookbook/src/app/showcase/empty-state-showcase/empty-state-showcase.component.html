Message State provides feedback about the application, content or workflow state to the user while, if possible, giving constructive guidance about next steps.

Notice

Empty State redirects here as the concept has been renamed Message State . The component itself, when used in the template, is still called kirby-empty-state and awaits renaming.

Message Types

The message state supports three different types of messaging with the themeColor input. For each of these, an icon that supports the message should be set with iconName .

  • Success - for positive messages like confirmations or receipts on longer flows
  • Warning - for warnings, errors or something the user should be particularly aware of
  • Empty - for communicating something that is empty, not configured or not used yetâ€”like an inbox without any messages

Simple

If setting a decorative icon does not add any value to the message, or if space is constrained, a simple variant can be used. It has a title and subtitle, and should be supported by a call to action button.

Please note: The message state expands horizontally to fill its container, and the title and subtitle area grows with it. If text needs to break differently a newline can be added with \n if the text comes from an expression or &#10; in case it is written directly in the template.

Buttons

If you add buttons, the following guidelines apply:

  • A primary button should resolve the state for the user
  • Any secondary buttons should present the user with further guidance or alternative options for resolving the state

If the first button has attention level 1, that level will be preserved. This is the button that should aim to resolve the state for the user. Any additional buttons will be adjusted to attention level 3 regardless of their initial level.

Accessibility

If you are creating custom alerts by showing a Message State inside a modal , the Message State title is automatically used as a label to add context for assistive technologies. For example, screen readers use the label to announce when the modal dialog appears and to give proper context when interacting with the content within.
